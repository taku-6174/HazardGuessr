Hazard Guesser
Hazard Guesser は、ICT技術を用いて防災教育をアップデートする、Windows向けの体験型避難シミュレーションシステムです。
単なるゲームではなく、気候変動に伴う水害リスクの増大という社会課題に対し、アルゴリズムとシミュレーションを用いて「最適な避難行動」を学習するためのツールとして開発されました。

開発背景と目的
地球温暖化の影響により、浸水や津波といった水害は激甚化・頻発化しています。
本プロジェクトは、従来の静的なハザードマップでは伝わりにくい「時間の経過とともに迫る脅威」をICTで可視化することを目的としています。
ユーザーに「自ら逃げ道を考える」体験を提供することで、環境変化への適応力を養います。

ゲーム詳細および機能
1. インタラクティブな避難ルート決定
ユーザーはドット絵で描かれた26種類の異なる地形マップ上で、現在地から指定された避難所までのルートを直感的に描画します。
単に道をなぞるだけでなく、ハザードマップの知識を基に「どこが浸水しやすいか」を予測する思考力が試されます。

2. リアルタイム・ディザスターシミュレーション
ルート決定後、シミュレーションが開始されると時間の経過とともに水が広がります。
浸水の広がりは視覚的に再現され、ユーザーが選んだルートが実際に安全であったかを動的に検証します。

3. 多角的なスコア算出システム
避難完了後、以下の3つの観点から論理的な採点を行い、ユーザーにフィードバックを提供します。

最短ルートの選定: 幅優先探索（BFS）を用いた正解ルートとの比較。

生存の可否: 浸水エリアに飲み込まれずに避難できたかの判定。

判断の迅速さ: 20秒以内のルート決定を基準としたタイムボーナス。

技術的特徴
ICTを活用した課題解決
幅優先探索 (BFS) の応用: ネットワーク経路探索等で使用されるアルゴリズムを避難ルート解析に応用し、システム側で常に論理的な正解を把握しています。

Pythonによる画像解析パイプライン: Canvaで作成した視覚的なマップ画像をPythonで数値配列に自動変換し、多種多様なステージの効率的な生成を実現しました。

非同期処理によるリアルタイム描画: C#（WPF）のタスク処理を活用し、シミュレーション計算とUIの描画を分離することで、滑らかなアニメーションを実現しています。
